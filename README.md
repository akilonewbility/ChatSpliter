# ChatSpliter

[中文](README.md) | [English](README_EN.md)

一款纯客户端 Minecraft Fabric 模组，通过关键词筛选将原版聊天窗口分离为多个独立的 HUD，每个 HUD 均可完全自定义。

## 功能

- **关键词筛选** — 根据关键词将聊天消息分流到独立的 HUD 窗口
- **三种匹配模式** — 任一关键词 / 全部关键词 / 正则表达式
- **完全可自定义** — 每个过滤器组独立配置位置、大小、缩放、不透明度、颜色等
- **文字缩放** — 按窗口独立缩放文字和布局
- **文字对齐** — 居左或居右对齐
- **滚动方向** — 底部向上（原版）/ 顶部向下（新消息在顶）/ 顶部锚定（自动上移填补空位）
- **原版风格淡出** — 消息根据可配置的显示时间和淡出时间渐出消失
- **自动换行** — 超长文本自动按窗口宽度换行
- **聊天历史** — 按 `T` 打开聊天栏后可在分窗口内滚动查看历史消息
- **拖动调整** — 打开聊天栏（`T`）后拖动窗口边框即可移动位置或调整大小
- **隐藏匹配消息** — 可选择从原版聊天栏中移除已匹配的消息
- **一键设置** — 打开聊天栏后每个窗口显示 ⚙ 齿轮按钮，点击直接跳转到该组的设置
- **可自定义快捷键** — 设置界面的快捷键显示在原版"控制 → 杂项"中

## 运行要求

- Minecraft **1.21.8**
- Fabric Loader **>= 0.16.9**
- Java **21+**
- 无需其他依赖（纯 Mixin 实现，不依赖 Fabric API）

## 使用方式

| 操作 | 方法 |
|------|------|
| 打开设置 | 按 `K` 键（可在 控制 → 杂项 中修改） |
| 查看聊天历史 | 按 `T` 打开聊天栏，在分窗口内滚动 |
| 移动窗口 | 按 `T`，拖动窗口标题区域 |
| 调整大小 | 按 `T`，拖动窗口右下角 |
| 打开组设置 | 按 `T`，点击窗口右上角 ⚙ 按钮 |

## 配置

配置文件保存在 `.minecraft/config/chatspliter.json`。

### 全局设置

| 设置项 | 说明 |
|--------|------|
| 启用模组 | 模组总开关 |
| 从主聊天隐藏已匹配消息 | 开启后匹配的消息不会出现在原版聊天栏 |

### 过滤器组设置

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 组名称 | `New Group` | 显示名称 |
| 关键词 | `[]` | 逗号分隔的匹配关键词 |
| 匹配模式 | 任一关键词 | `任一关键词` / `全部关键词` / `正则表达式` |
| X / Y 坐标 | `2, 2` | 窗口位置（像素） |
| 宽度 / 高度 | `320, 180` | 窗口大小（像素） |
| 文字缩放 | `1.0` | 文字和布局的整体缩放比例 |
| 总显示时间 | `15` | 消息从出现到完全消失的秒数 |
| 淡出时间 | `10` | 淡出效果持续的秒数（在显示时间的末尾） |
| 背景不透明度 | `0.5` | 文字背景的暗度（0–1） |
| 文字不透明度 | `1.0` | 文字可见度（0–1） |
| 行间距 | `0` | 消息行之间的额外像素 |
| 最大显示行数 | `100` | 内存中缓存的最高消息行数 |
| 显示时间戳 | 关 | 在每条消息前显示 `HH:mm:ss` 格式时间戳 |
| 显示窗口标题 | 开 | 在窗口右上角显示组名称 |
| 文字对齐 | 居左 | `居左` / `居右` |
| 滚动方向 | 底部向上 | `底部向上` / `顶部向下` / `顶部锚定` |

### 滚动方向模式

| 模式 | 行为 |
|------|------|
| **底部向上** | 原版风格 — 最新消息在底部 |
| **顶部向下** | 与原版相反 — 最新消息在顶部 |
| **顶部锚定** | 旧消息在顶部，新消息向下延伸；旧消息渐出消失后，新消息自动上移填补空位 |

## 默认过滤器组

首次启动时模组会创建三个示例组：

| 组 | 关键词 |
|----|--------|
| Party | `party` |
| Whisper | `whispers`, `whisper to you` |
| System | `[System]`, `[Server]` |

## 构建

```bash
./gradlew build
# 产物位于 build/libs/chatspliter-1.0.0.jar
```

在 VS Code 中构建：安装 **Extension Pack for Java** 和 **Gradle for Java**，打开项目文件夹后运行 `build` Gradle 任务。
